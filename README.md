# 量化多因子选股策略

基于 **pybroker** 回测框架的 A 股中证 500 多因子选股轮动策略。

---

## 策略概要

| 项目 | 说明 |
|------|------|
| **选股范围** | 中证 500 指数成分股（约 500 只） |
| **数据来源** | baostock（日 K 线 + 估值）+ akshare（成分股） |
| **回测框架** | pybroker 1.2（官方轮动交易模式） |
| **数据缓存** | 本地 SQLite 数据库，增量下载，不重复请求 |
| **持仓模式** | 纯多头，每日轮动 |
| **交易逻辑** | T 日收盘打分 → T+1 日开盘卖出淘汰 / 买入新入选 |

### 多因子模型（5 因子等权）

| 因子 | 含义 | 权重 |
|------|------|------|
| 盈利收益率 | 1 / PE_TTM，越高估值越低 | 25% |
| 净资产收益率 | 1 / PB_MRQ，越高估值越低 | 20% |
| 20 日动量 | 近 20 个交易日涨跌幅 | 20% |
| 量比 | 当日成交量 / 20 日均量 | 15% |
| RSI 信号 | RSI 接近超卖区得分高（反转） | 20% |

## 快速开始

### 默认运行（最近两年回测）

```bash
python 量化多因子选股策略.py
```

### 指定回测区间

```bash
python 量化多因子选股策略.py --start 2023-01-01 --end 2025-12-31
```

### 快速测试（只用 30 只股票，每日持仓 10 只）

```bash
python 量化多因子选股策略.py --stocks 30 --top-n 10 --cash 500000
```

### 查看所有参数

```bash
python 量化多因子选股策略.py --help
```

---

## 命令行参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--start` | str | 两年前 | 回测起始日期 |
| `--end` | str | 昨天 | 回测结束日期 |
| `--stocks` | int | 0（全部） | 下载股票数量上限，0=全部 500 只 |
| `--top-n` | int | 5 | 每日持仓股票数量 |
| `--cash` | int | 1000000 | 初始资金（元） |

> 数据区间会自动比回测区间早 90 天，用于技术指标预热。

## 运行流程

```
步骤 1 ─ 获取中证 500 成分股（akshare）
步骤 2 ─ 下载/加载日 K 线数据（baostock → SQLite）
步骤 3 ─ 计算技术指标（RSI, 动量, 量比, 均线, 波动率）
步骤 4 ─ 多因子打分与每日截面标准化
步骤 5 ─ pybroker 回测（轮动交易）
步骤 6 ─ 输出回测指标（交易次数, 收益率, 最大回撤, 夏普比率等）
步骤 7 ─ 生成 4 象限可视化图表（净值 / 回撤 / 月度热力图 / 统计）
```

## 数据说明

| 项目 | 来源 | 覆盖范围 |
|------|------|----------|
| 日 K 线（OHLCV） | baostock `query_history_k_data_plus` | 1990 年至今 |
| 估值（PE/PB/PS） | baostock（同上接口） | 同上 |
| 成分股列表 | akshare `index_stock_cons` | 当前最新成分股 |

### 关于历史成分股

当前策略使用 **最新** 中证 500 成分股列表。部分新股在较早的历史区间没有行情数据（尚未上市），属于正常现象，程序会自动标记跳过，不影响回测。

### 缓存机制

首次运行会通过 baostock 下载全部行情数据到本地 `stock_kline_cache.db`（约 3-5 分钟）。之后运行只下载差值（新增日期 / 新股票），已有数据秒级加载。支持跨日期范围复用——换一个 `--start` 不会重新下载已缓存的部分。

## 已知限制

- 使用当前成分股而非历史时点成分股（免费数据源限制）
- 不包含分红、送转等复权因子的独立校验
- pybroker 1.2 版本的 `before_exec` API 与官网文档有不一致之处，已做适配
